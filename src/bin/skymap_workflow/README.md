# Sphradiometer Skymap DAG Workflows

This directory contains an exploratory, config-driven workflow split into two
DAGs.

1. Precompute once for a PSD:

```bash
./sph_skymap_precompute_pipe.py example_precompute.ini
condor_submit_dag precompute_skymap.dag
```

The precompute DAG is written in the same directory as the INI file.  This
creates a reusable precompute directory under
`outputs/precalc/<IFOs>_<mode>_<precalc_len>/` containing
sphradiometer projection
objects for the detector set, PSD weighting convention, `precalc_len`, and
sample rate.  The precompute DAG is split by detector combination, so for
`H1,L1,V1` it creates independent nodes for `H1`, `L1`, `V1`, `H1L1`, `H1V1`,
`L1V1`, and `H1L1V1`.

2. Generate skymaps for injections or noise candidates from a GstLAL run:

```bash
./sph_skymap_injection_pipe.py example_injections.ini
condor_submit_dag injection_skymaps.dag
```

The injection INI only needs the GstLAL run directory and injection SQLite:

```ini
[gstlal]
run_dir = /path/to/gstlal/run
injection_database = /path/to/run/H1L1V1-ALL_LLOID_bbh_injections-START-DUR.sqlite
frame_cache =
```

The pipe discovers the frame cache, segments file, detector channels,
reference PSDs, injection XML, SVD bank groups, and
`gstlal_inspiral_injection_template_match` files.  It writes
`outputs/bank_row_map.csv` and `outputs/autogenerated_manifest.csv`, then
creates one `gstlal_inspiral_calc_snr` node and one skymap node per selected
manifest row.

For non-injection candidates, use a non-injection SQLite database and set
`database_type = noise`:

```ini
[gstlal]
run_dir = /path/to/gstlal/run
database_type = noise
noise_database = /path/to/run/H1L1V1-ALL_LLOID-START-DUR.sqlite
frame_cache =

[manifest]
order_by = far
order = desc
limit = 100
instruments = H1,L1,V1
zero_lag_only = true
exclude_time_windows = center:1243533585:300
far_gt = 3e-7
```

Noise mode queries GstLAL `coinc_def_id = 0` candidates, resolves the SVD bank
row from the candidate's stored template identifiers, chooses the reference PSD
nearest the candidate time, and writes the same `outputs/autogenerated_manifest.csv`
format.  With `zero_lag_only = true`, zero lag is selected from the offsets in
the `time_slide` table, not from the numeric `time_slide_id`.  It deliberately
does not pass injection XML or injection time slides to `gstlal_inspiral_calc_snr`.
For noise-like studies, use a lower FAR bound such as `far_gt = 3e-7` Hz so very
significant zerolag candidates are not treated as noise.

Leave `[gstlal] frame_cache` blank to use `<run_dir>/frame.cache`.  Set it
explicitly when the skymap rerun should use a different frame cache.

Each DAG directory is self-contained:

```text
<injection-config-dir>/
  injection_skymaps.dag
  injection_skymaps.sh
  *.sub
  logs/
  outputs/
    calc_snr/
    coeff/
    skymap/
    skymap_branch/
```

The precompute DAG follows the same layout, with precomputed objects under
`outputs/precalc/`.

After a skymap DAG has produced FITS files, summarize the completed maps with
paper-style injection diagnostics:

```bash
sph_skymap_injection_summary.py \
  --manifest outputs/autogenerated_manifest.csv \
  --skymap-dir outputs/skymap \
  --output-dir outputs/summary \
  --missing-ok
```

This writes `outputs/summary/summary.csv`, a Fig. 3-style P-P plot, a Fig.
4-style searched-area distribution, and, when matching BAYESTAR FITS maps are
provided, a Fig. 5-style searched-area ratio distribution:

```bash
sph_skymap_injection_summary.py \
  --manifest outputs/autogenerated_manifest.csv \
  --skymap-dir outputs/skymap \
  --bayestar-dir outputs/skymap \
  --bayestar-pattern 'sim{simulation_id}_bayestar.fits' \
  --output-dir outputs/summary \
  --missing-ok
```

To generate BAYESTAR comparison maps as part of the injection DAG, enable the
optional `[bayestar]` section:

```ini
[bayestar]
enabled = true
write_png = true
f_final = 1024
f_low = 30
f_high_truncate =
snr_series_half_width = 0.1
omp_num_threads =
waveform =
prior_distance_power =
rescale_loglikelihood =
disable_snr_series = false
keep_going = true
```

The pipe creates a shared `outputs/bayestar/input_with_f_final.sqlite` copy of
the GstLAL database, fills missing `sngl_inspiral.f_final` values, and schedules
one `sph_bayestar_from_sqlite.py` node per manifest row after that row's
`gstlal_inspiral_calc_snr` node.  The BAYESTAR helper attaches the dumped
complex SNR time series to the event before localizing it, so BAYESTAR does not
fall back to a zero-noise autocorrelation SNR series.  The comparison FITS files
are written as `outputs/bayestar/sim<simulation_id>_bayestar.fits`.
If `omp_num_threads` is left empty, the helper uses `[condor] request_cpus` for
`OMP_NUM_THREADS` so BAYESTAR does not oversubscribe execute nodes.
Summarize them with:

```bash
sph_skymap_injection_summary.py \
  --manifest outputs/autogenerated_manifest.csv \
  --skymap-dir outputs/skymap \
  --bayestar-dir outputs/bayestar \
  --bayestar-pattern 'sim{simulation_id}_bayestar.fits' \
  --output-dir outputs/summary \
  --missing-ok
```

## Condor container config

The `[condor]` section follows the same convention used by the local
`ligo.skymap` lensing DAG configs:

```ini
[condor]
cluster = resceubbc
universe = vanilla
accounting_group = ligo.dev.o5.cbc.uber.gstlaloffline
accounting_group_user = alvin.li
request_cpus = 2
request_memory = 6GB
request_disk = 4GB
container_mode = auto
container_image = /data/alvin.li/debian-trixie-gstlal-260630.sif
container_bind_paths = /data:/data,/home:/home
extra_environment =
```

`container_mode = auto` uses native HTCondor Singularity support when
`container_image` is set, and disables containers when it is empty.  Use a SIF
or other image path visible on execute nodes; do not rely on a local writable
sandbox path such as `/localscratch/...` from the submit host.

## Manifest

The injection XML contains injection time and sky position, but not the SVD
`bank_number` and `row_number` required by `gstlal_inspiral_calc_snr`.  For now
the injection DAG therefore consumes a manifest CSV with one row per skymap:

```text
simulation_id,coinc_event_id,center_time,gps_start_time,gps_end_time,snr_start,snr_end,bank_number,row_number,row_counts,ht_gate_threshold,ra_deg,dec_deg,pre_trigger
```

For manual studies, a manifest can still be generated from a recovered-trigger
SQLite database with GstLAL's diagnostic helper:

```bash
gstlal_inspiral_sqlite_injection_manifest manifest_from_sqlite.ini
```

The manifest generator reads GstLAL's injection-association coincidence rows,
orders recovered injections by FAR ascending by default, and can apply
threshold cuts such as `far_lt`, `snr_gt`, `max_time_offset`, or an arbitrary
SQL `where` expression.  For localization validation, set `max_time_offset` to
reject recovered triggers whose end time is too far from the simulated
geocentric end time; otherwise low-SNR association rows can select nearby noise
triggers that are not valid tests of whether the skymap contains the injection
position.  Set `order_by = snr` and `order = desc` to inspect the loudest
recoveries first, or set `limit` to keep only the first N rows after sorting
and cuts.

The final GstLAL SQLite database does not store the SVD `bank_number` and
`row_number` needed by `gstlal_inspiral_calc_snr`.  In `[gstlal]` auto mode,
the pipe derives them from `gstlal_inspiral_injection_template_match` outputs
by matching injections on geocentric time and then locating the matched
template in the SVD bank group.  The resulting manifest carries row-specific
`svd_bank`, `bank_number`, `row_number`, and `reference_psd` values.

In `[gstlal] database_type = noise` auto mode, no injection-template-match
files are needed.  The pipe uses GstLAL's candidate SQLite rows and the SVD
bank XML files in the run directory to construct the same per-row fields.

If the manifest contains an `injection_xml` column, that per-row file is passed
to `gstlal_inspiral_calc_snr`.  This is preferred for dense injection sets:
split the injection XML into one file per injection, then list the corresponding
single-injection XML in each manifest row.  If the column is absent or empty,
the global `[injections] injection_xml` value from the INI file is used.

## Notes

- `precalc_dir` is reusable only when the PSD, detector set, weighting mode,
  `precalc_len`, `delta_t`, and effective sample rate are unchanged.
- For precompute configs, `[precompute] precalc_dir` is optional.  Leave it
  unset unless you intentionally want to reuse or overwrite a specific
  precompute directory.
- Keep only detector-combination directories inside `precalc_dir`; metadata is
  written beside it as `<precalc_dir>.metadata/` because
  `RapidLocalization.read()` scans every entry in `precalc_dir`.
- The default weighting mode is `asd`, using relative detector ASD weights
  normalized by the network median per frequency bin.
- New precompute jobs default to `apply_projection = false`, which stores the
  paper-style delay basis without freezing the antenna/PSD projection at the
  precompute epoch.  Harmonic skymap jobs that read such a delay-only precompute
  automatically apply the full GMST-dependent projection at event time using the
  PSD metadata written beside the precompute directory.
- The default skymap backend is `coherent_likelihood`.  It evaluates the
  detector-delay CBC likelihood used as the reference in the diagnostic studies
  and writes both probability maps and spherical harmonic coefficient FITS
  files by default.  Single-event `sph_skymap_from_calc_snr.py` runs also
  default to this backend.
- The coherent-likelihood backend evaluates the likelihood on the requested HEALPix
  grid and on the Gauss-Legendre spherical-harmonic coefficient grid with a
  vectorized detector-delay and antenna-response implementation.  Leave
  `time_prior_step` unset for the sample-spaced time grid; setting it to a
  coarse value speeds up jobs but can move the peak for narrow two-detector
  likelihoods.  Production-like injection studies should use
  `time_prior_center = center_time` and `time_prior_half_width = auto`, so the
  map is conditioned only on the reported trigger time and dumped SNR interval.
  Use `time_prior_center = truth` only for controlled diagnostics.  The default
  coherent path uses `paper_snr_window = full`, which evaluates the statistic
  over the full dumped SNR time series rather than a short truth-centered window.
  For one-detector events it falls back to the detector antenna power pattern,
  because timing triangulation is unavailable.
- The coherent-likelihood backend defaults to `amplitude_model = paper_cross`.
  This evaluates the regulated paper statistic directly: β = ±1 CBC projection,
  ψ = 0, and cross-detector terms only.  It deliberately avoids the auto-power
  terms that made the full coherent score too sharp in diagnostic injections.
  `amplitude_model = cbc` is available as a diagnostic inclination/polarization
  grid over the full coherent score, while `amplitude_model = two_pol` maximizes
  over arbitrary complex plus/cross amplitudes.  The old pure circular branch
  statistic is still available with `amplitude_model = circular`; it is useful
  for regression tests but can confidently choose the wrong sky mode for generic
  CBC inclinations.
- The coherent-likelihood score can be tempered with `direct_score_scale`.  A
  value of `-1` or any non-positive value selects the default paper-cross FFT
  normalization, `1 / SNR-window length`; positive values are explicit
  calibration overrides for empirical containment studies.
- Strict paper-cross runs should start with `containment_calibration = false`,
  `direct_score_scale = -1`, `direct_smoothing_deg = 0`, `antenna_mixture = 0`,
  `isotropic_mixture = 0`, and `peak_time_sigma = 0`.  This applies the same
  regulated cross-detector statistic to every event, independent of SNR.
- `peak_time_sigma` enables an optional detector peak-time consistency term.
  When enabled, it is gated by the measured SNR time series rather than by the
  event's network SNR by default: every detector must pass
  `peak_time_min_peak_abs` and, if configured, `peak_time_min_prominence`.  This
  keeps noisy low-SNR peak times from forcing an overconfident wrong arc.
- Fixed-grid coherent-likelihood maps can also be made conservative with
  `isotropic_mixture`, which mixes a configured fraction of uniform sky
  probability into the normalized map.  This is a calibration layer on top of the
  paper statistic, not a replacement likelihood.
- `direct_smoothing_deg` applies a Gaussian smoothing kernel to fixed-grid
  coherent-likelihood probability maps before the isotropic mixture is applied.
  This is intended for containment calibration studies, especially near the
  low-SNR limit where the paper-cross peak is noise dominated.
- `antenna_mixture` mixes in the network antenna-response probability.  This is
  a deliberately weak low-SNR fallback: it uses detector sensitivity on the sky
  but does not introduce Bayestar-style distance, phase, or CBC amplitude
  marginalization.
- Set `containment_calibration = true` only for empirical containment studies.
  That opt-in mode passes each candidate's network SNR and can deliberately
  broaden near-threshold maps (`low_snr_score_scale`, `low_snr_smoothing_deg`,
  `low_snr_antenna_mixture`) while sharpening loud events toward the configured
  high-SNR settings.  It is not part of the strict paper-cross baseline.
- Set `[skymap] adaptive = true` to write the coherent-likelihood map as a
  multi-order HEALPix/NUNIQ FITS file.  Adaptive jobs start from `direct_nside`
  and refine high-density pixels through `adaptive_nside_stop`, bounded by
  `adaptive_max_evals`.  Adaptive output is the combined probability map only;
  fixed-grid p/n branch maps and spherical harmonic coefficient products are not
  declared for adaptive DAG jobs.
- The `harmonic` backend is the native fast sphradiometer harmonic correlator.
  It is kept under that name for upstream clarity and regression tests.
  Diagnostic injections show that this native cross-power statistic can place
  confident peaks far from the true location for the CBC SNR inputs used here.
  Rows with fewer than 3 detectors are routed away from `harmonic` through
  `low_ifo_backend = coherent_likelihood` by default unless explicitly disabled.
- Harmonic skymap jobs using older projected precompute directories do not apply
  runtime GMST projection correction unless `[skymap] runtime_gmst_correction =
  true` is set explicitly.  For new delay-only precompute directories, full
  runtime projection is enabled automatically.
- The per-injection skymap jobs keep H1, L1, and V1 if their SNR XML files are
  present.  Weak detectors are downweighted by the precomputed PSD/ASD weights,
  not dropped.
- The skymap jobs write the combined probability map in `outputs/skymap/`, the
  two beta-branch maps in `outputs/skymap_branch/`, and the raw spherical
  harmonic coefficients in `outputs/coeff/` by default.  Set
  `[skymap] write_coefficients = false` to omit the coefficient products in DAG
  workflows.  For single-event `sph_skymap_from_calc_snr.py` runs, coefficient
  output defaults to `<output>_coeffp.fits` and `<output>_coeffn.fits`; pass
  `--no-output-coefficients` to disable it.
